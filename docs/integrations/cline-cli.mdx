## Install

Install the [Cline CLI](https://docs.cline.bot/usage/cli-overview). For the IDE extension, see [Cline](/integrations/cline).

```bash
npm install -g cline
```

<Note>If Cline CLI is not installed and `npm` is available, `lychee launch cline` will prompt to install `cline@latest`.</Note>

## Usage with Lychee

### Quick setup

```bash
lychee launch cline
```

When launched through `lychee launch cline`, Lychee sets Cline's provider to Lychee, points it at the local Lychee endpoint, and selects the model you choose.

To configure without launching:

```shell
lychee launch cline --config
```

### Run directly with a model

```shell
lychee launch cline --model qwen3.5
```

To use a cloud model:

```shell
lychee launch cline --model kimi-k2.6:cloud
```


### Pass a prompt to Cline

Arguments after `--` are passed directly to Cline:

```shell
lychee launch cline -- "summarize this repository"
```

To open Cline's Kanban board:

```shell
lychee launch cline -- kanban
```

### Manual setup

To configure Cline CLI manually, first make sure Lychee is running and the model you want to use is available:

```shell
lychee pull qwen3.5
```

Then run Cline's interactive auth flow:

```shell
cline auth
```

Select Lychee as the provider, use `http://localhost:11434` as the base URL if prompted, and choose a model such as `qwen3.5` or `kimi-k2.6:cloud`.

To check the current Cline configuration:

```shell
cline config
```

To start an interactive session:

```shell
cline
```
