Documentation

Everything you need to get Lychee up and running — from installation to advanced configuration.

Quick Start

Get up and running with Lychee in under a minute. The fastest way is a single install command:

💡
Tip: Lychee requires macOS 11+, Ubuntu 20.04+, or Windows 10/11. GPU acceleration is automatic when available.
BASH
# Install Lychee
curl -fsSL https://lychee.ai/install.sh | sh

# Pull a model
lychee pull llama3.1:8b

# Start chatting
lychee run llama3.1:8b

That's it. Lychee will download and start the model. You can now chat in the terminal, open the app UI, or use the REST API at http://localhost:11434.

Installation

Choose your platform below for detailed installation instructions.

# Option 1: Download the macOS app (Apple Silicon + Intel)
# Visit: https://lychee.ai/download

# Option 2: Install via Homebrew
brew install lychee

# Option 3: Install script
curl -fsSL https://lychee.ai/install.sh | sh

On Apple Silicon Macs, Lychee uses Metal GPU acceleration automatically for maximum performance. No setup required.

# Install script (recommended)
curl -fsSL https://lychee.ai/install.sh | sh

# Or install manually
wget https://lychee.ai/releases/latest/linux-amd64/lychee
chmod +x lychee
sudo mv lychee /usr/local/bin/

# Enable as a system service
sudo systemctl enable lychee
sudo systemctl start lychee
ℹ️
NVIDIA GPU: Install CUDA 11.8+ for GPU acceleration. AMD GPU: ROCm 5.6+ is supported on Linux.
# Option 1: Download Windows installer (.exe)
# Visit: https://lychee.ai/download#windows

# Option 2: winget
winget install Lychee.Lychee

# Option 3: Scoop
scoop install lychee

# Verify installation
lychee --version

Windows requires WSL2 or native installation. NVIDIA CUDA and AMD ROCm are supported. Run PowerShell as Administrator for the first install.

# CPU only
docker pull lychee/lychee:latest
docker run -d -v lychee:/root/.lychee -p 11434:11434 --name lychee lychee/lychee

# NVIDIA GPU
docker run -d --gpus=all -v lychee:/root/.lychee \
  -p 11434:11434 --name lychee lychee/lychee

# Test the API
curl http://localhost:11434/api/tags

Running Models

Pull a model

Download a model from the Lychee registry. The download happens in the background with resume support.

BASH
lychee pull llama3.1:8b

# Pull a specific quantization
lychee pull llama3.1:8b-q4_0

# Pull from HuggingFace directly
lychee hf pull bartowski/Meta-Llama-3.1-8B-Instruct-GGUF

Run a model

Start an interactive chat session in your terminal.

BASH
lychee run llama3.1:8b

# Pass a prompt directly (one-shot)
lychee run llama3.1:8b "Explain quantum entanglement"

# Pipe input
cat document.txt | lychee run llama3.1:8b "Summarize this:"

List local models

BASH
lychee list

API Reference

Lychee runs a local REST API on http://localhost:11434. It is fully compatible with the OpenAI API spec, so any OpenAI SDK works out of the box by setting base_url.

ℹ️
OpenAI compatibility: Use /v1/chat/completions, /v1/embeddings, and /v1/models endpoints with any OpenAI SDK. Set OPENAI_BASE_URL=http://localhost:11434/v1 and OPENAI_API_KEY=lychee.
Method Endpoint Description
POST /api/generate Generate a completion for the given prompt
POST /api/chat Generate a chat completion with message history
GET /api/tags List all locally installed models
POST /api/pull Pull a model from the Lychee registry
POST /api/push Push a model to the Lychee registry
POST /api/embeddings Generate embeddings for the given input
POST /api/create Create a model from a Modelfile
POST /api/copy Copy a model to a new name
DELETE /api/delete Delete a local model
POST /api/show Show information about a model
GET /api/ps List currently loaded models in memory
GET /api/version Get the Lychee server version

Example: Chat Completion

CURL
curl http://localhost:11434/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "model": "llama3.1:8b",
    "messages": [
      { "role": "system", "content": "You are a helpful assistant." },
      { "role": "user", "content": "Why is the sky blue?" }
    ],
    "stream": true
  }'

Example: Generate Embeddings

CURL
curl http://localhost:11434/api/embeddings \
  -d '{
    "model": "nomic-embed-text",
    "prompt": "The quick brown fox jumps over the lazy dog"
  }'

lychee embed

Generate vector embeddings for one or more text inputs. Outputs to text, JSON, or CSV. Reads from stdin when no arguments are given — perfect for piping into RAG pipelines.

💡
Requires: an embedding model loaded locally. Pull nomic-embed-text (the default) with lychee pull nomic-embed-text.

Usage

BASH
lychee embed [text...] [flags]

Flags:
  -m, --model     nomic-embed-text   Embedding model to use
  -o, --output    text               Output format: text, json, csv
      --truncate  true               Truncate input if over context length

Examples

BASH
# Embed a single string
lychee embed "hello world"

# Embed multiple inputs
lychee embed "cat" "dog" "fish"

# JSON output for downstream processing
lychee embed "semantic search query" --output json

# Pipe a file
cat document.txt | lychee embed --model nomic-embed-text

# Use a custom model
lychee embed "hello" --model mxbai-embed-large --output csv

JSON Output Schema

JSON
{
  "model": "nomic-embed-text",
  "embeddings": [
    [-0.0142, 0.0478, -0.0215, ...]  // 768 floats for nomic-embed-text
  ]
}

lychee compare

Run two models on the same prompt simultaneously and display a side-by-side comparison of their outputs, speed, and response length. Useful for picking the right model for a specific task.

Usage

BASH
lychee compare <model1> <model2> [flags]

Flags:
  -p, --prompt        Custom prompt (default: three laws of robotics)
      --system         System message for both models
      --max-tokens     Max tokens per response (default: 512)
      --side-by-side   Wide-terminal side-by-side layout

Examples

BASH
# Compare two models with the default prompt
lychee compare llama3.2:3b mistral:7b

# Use a custom prompt
lychee compare llama3.1:8b qwen3:8b \
  --prompt "Explain transformer attention in one paragraph"

# With a system message
lychee compare phi4:14b deepseek-r1:14b \
  --system "You are a concise technical writer" \
  --prompt "What is gradient descent?"

Summary Output

  ──────────────────────────────────────────────────────────────
  Speed                38.2 tok/s    22.1 tok/s    llama3.2:3b (faster)
  Response length      412 chars     587 chars     mistral:7b (longer)

lychee stats

A live dashboard that shows all models currently loaded in memory. Refreshes every second (configurable). Press Ctrl+C to exit.

Usage

BASH
lychee stats [flags]

Flags:
  --interval   Refresh interval in seconds (default: 1)

Example Output

OUTPUT
  Lychee Stats  [14:22:07]  Ctrl+C to exit
  ────────────────────────────────────────────────────────────────────────────
  MODEL                        VRAM       RAM        CTX        EXPIRES
  ────────────────────────────────────────────────────────────────────────────
  llama3.1:8b                  4.7 GB     ─          8k         4m32s
  nomic-embed-text             274.0 MB   ─          ─          2m18s
ℹ️
The dashboard refreshes in-place using ANSI escape codes. Works in any modern terminal on macOS, Linux, and Windows Terminal.

Environment Variables

Configure Lychee behavior through environment variables. These can be set in your shell profile or passed directly when starting the server.

VariableDefaultDescription
LYCHEE_HOST 127.0.0.1:11434 Host and port for the Lychee server. Set to 0.0.0.0:11434 to expose on the network.
LYCHEE_MODELS ~/.lychee/models Directory where downloaded models are stored.
LYCHEE_DEBUG false Enable verbose debug logging. Useful for troubleshooting.
LYCHEE_HF_TOKEN — HuggingFace access token for downloading gated/private models.
LYCHEE_KEEP_ALIVE 5m How long to keep models loaded in memory after last use. Set to -1 to keep forever.
LYCHEE_NUM_PARALLEL 1 Number of parallel inference requests. Increase for multi-user server deployments.
LYCHEE_MAX_LOADED_MODELS 3 Maximum number of models to keep loaded in memory simultaneously.
LYCHEE_FLASH_ATTENTION false Enable Flash Attention for faster inference (requires compatible GPU).

Modelfile

A Modelfile lets you create custom models with specific system prompts, parameters, and templates. Think of it like a Dockerfile for AI models.

Syntax Reference

InstructionDescription
FROMBase model to build on (required). Can be a local model or registry reference.
SYSTEMSystem prompt injected at the start of every conversation.
PARAMETERSet model parameters: temperature, top_p, top_k, num_ctx, num_predict, repeat_penalty.
TEMPLATECustom prompt template (Go template syntax). Overrides the default chat template.
MESSAGEAdd example few-shot messages (user/assistant pairs) to the context.
ADAPTERApply a LoRA adapter file to the base model.
LICENSESpecify the model license for distribution.

Example Modelfile

MODELFILE
# Build from any base model
FROM llama3.1:8b

# Set a custom system prompt
SYSTEM """
You are Aria, a helpful senior software engineer at Lychee.
You write clean, well-documented code and explain your reasoning.
Always use best practices and consider edge cases.
"""

# Tune model parameters
PARAMETER temperature 0.7
PARAMETER top_p 0.9
PARAMETER num_ctx 8192
PARAMETER repeat_penalty 1.1

# Add few-shot example
MESSAGE user "How do I reverse a string in Python?"
MESSAGE assistant "You can use slicing: `s[::-1]`. For clarity in production code, consider `''.join(reversed(s))`."
BASH
# Create the custom model
lychee create aria -f ./Modelfile

# Run it
lychee run aria

Integrations

Lychee works with many popular tools via its OpenAI-compatible API. Set your base URL and a dummy API key:

PYTHON
# Using OpenAI Python SDK with Lychee
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="lychee",  # required but not used
)

response = client.chat.completions.create(
    model="llama3.1:8b",
    messages=[{"role": "user", "content": "Hello!"}]
)
print(response.choices[0].message.content)

GPU Setup

GPU acceleration is detected and enabled automatically. Here's what's supported:

HardwareDriver/SDK RequiredNotes
Apple Silicon (M1/M2/M3/M4)macOS 11+ (built-in Metal)Automatic, best performance on Mac
NVIDIA GeForce / RTXCUDA 11.8+ driversInstall latest Game Ready or Studio drivers
AMD Radeon (Linux)ROCm 5.6+Supported on RDNA2+ GPUs
Intel Arc / UHD (experimental)Intel GPU pluginBeta support, set LYCHEE_INTEL_GPU=true
CPU (fallback)NoneWorks on any machine, slower inference
⚠️
VRAM: Ensure you have enough VRAM for the model. A 7B Q4 model needs ~4GB, a 13B needs ~8GB, a 70B needs ~40GB. Use lychee ps to check current GPU memory usage.

Troubleshooting

Common issues and how to fix them.

Server not running

⚠️
Error: dial tcp 127.0.0.1:11434: connect: connection refused
BASH
# Start the server manually
lychee serve

# Or enable it as a system service (Linux)
sudo systemctl enable --now lychee

# Check if it's running
curl http://localhost:11434/api/version

Model not found

⚠️
Error: model "xyz" not found, try pulling it first
BASH
# Pull the model first
lychee pull llama3.1:8b

# Or pull from HuggingFace
lychee hf pull bartowski/Meta-Llama-3.1-8B-Instruct-GGUF

# List what you already have
lychee list

Out of VRAM / slow inference

BASH
# Check which models are loaded and how much VRAM they use
lychee stats

# Unload a model from memory
lychee stop llama3.1:8b

# Use a smaller quantization (less VRAM)
lychee hf pull bartowski/Llama-3.1-8B-Instruct-GGUF --quant q3_k_m

# Scan your hardware to find models that fit
lychee scan

Download stuck or corrupt

BASH
# Lychee downloads are resumable — just re-run to continue
lychee hf pull bartowski/Meta-Llama-3.1-8B-Instruct-GGUF

# If a file is corrupt, delete the .part file and retry
rm ~/.lychee/models/hf-models/**/*.part
lychee hf pull bartowski/Meta-Llama-3.1-8B-Instruct-GGUF

GPU not detected

BASH
# Enable debug logging to see GPU detection
LYCHEE_DEBUG=1 lychee serve

# NVIDIA: ensure drivers and CUDA are installed
nvidia-smi

# AMD Linux: verify ROCm
rocminfo | grep "Agent Type"

# Intel Arc: enable beta GPU support
LYCHEE_INTEL_GPU=1 lychee serve

FAQ

Is Lychee free? +

Yes. Lychee is free, open source (MIT license), and always will be. The models you download are also free — no subscriptions, no API keys, no usage limits.

Do I need a GPU? +

No. Lychee runs on CPU-only machines. GPU acceleration is automatic when available and makes inference significantly faster — but small models (1-3B) run fine on modern CPUs. Run lychee scan to see what fits your hardware.

Can I use Lychee with my existing OpenAI code? +

Yes. Lychee exposes an OpenAI-compatible API at http://localhost:11434/v1. Point any OpenAI SDK to that base URL with any dummy API key and it works. No code changes needed beyond swapping the base URL.

Where are models stored? +

Models are stored in ~/.lychee/models/ by default on all platforms. Override this with the LYCHEE_MODELS environment variable to point to a different drive or directory.

How is Lychee different from Ollama? +

Lychee adds direct HuggingFace pulls with resumable downloads and SHA256 verification, a 275+ model catalog, and four exclusive power-user commands: lychee embed, lychee scan, lychee compare, and lychee stats. The core inference engine is the same.

What model format does Lychee use? +

Lychee uses GGUF format models for local inference via llama.cpp. When pulling from HuggingFace, only .gguf files are downloaded. GGUF supports Q2–Q8 quantizations and FP16/BF16 for high-precision inference.

Can I run multiple models at the same time? +

Yes. Set LYCHEE_MAX_LOADED_MODELS (default: 3) to keep multiple models in memory simultaneously. Use lychee stats to monitor VRAM usage across all loaded models.
